# 生活科普寫文規則

## 核心定位

**日本「豆知識」風格**——短、輕鬆、讓人有「喔！原來我國中學過這個」的感覺。
目標讀者：台灣一般大眾（對應國中小課綱知識儲備）。

---

## 1. 文章長度

- 正文 **150–300 字**（不含 YAML front matter）
- 不夠 150 字：知識點還沒說完，繼續補充。
- 超過 300 字：切成兩篇，或刪去枝節。

---

## 2. 文章結構（三段式）

```
① 一個核心知識點（1–3 句）
   → 直接點出今天的主角，不繞彎子。

② 生活連結（3–6 句）
   → 說明這個知識點在哪裡出現、為什麼有感。
   → 用具體場景：「你有沒有發現…」、「下次去超市時…」

③ 一句收尾
   → 輕鬆、有點趣味，帶出「原來如此」的餘韻。
   → 不要下硬結論，也不要說教。
```

---

## 3. 語氣原則

- **親切，不說教**：像在跟朋友分享有趣小事，不是在上課。
- **主動句優先**：「水在 100°C 沸騰」優於「沸騰是水在 100°C 時發生的現象」。
- **用「你」拉近距離**：「你有沒有想過…」、「下次試試看…」
- **結尾不要「總結」**：不用寫「因此，我們可以知道…」這種句子。

---

## 4. 禁止事項

| 禁止 | 原因 |
|------|------|
| 艱深術語（未解釋即使用） | 讀者會直接跳走 |
| 複雜計算、公式推導 | 這裡是豆知識，不是參考書 |
| 學術引用（APA/MLA 格式） | 打破輕鬆氛圍 |
| 連續超過 3 句的長句 | 難以消化 |
| 「眾所周知」、「顯而易見」 | 如果眾所周知就不用寫了 |
| 英文縮寫未標注中文 | 例：DNA 要寫「DNA（去氧核糖核酸）」，除非極常見 |

---

## 5. 圖表規範

- **格式**：SVG（優先）或 PNG（解析度 ≥ 144 dpi）
- **存放路徑**：`science/[category]/assets/[slug]-[圖號].svg`
  - 例：`science/physics/assets/bernoulli-01.svg`
- **用途**：示意圖、簡單數線圖、比較表——不做複雜數據圖
- **風格**：簡潔線條，避免過多顏色（最多 3 色）
- **優先順序（尤其 Instagram 需要圖片才能發文）**：
  1. 自製圖表（SVG/PNG 示意圖）——首選
  2. AI 生成圖片——次選，風格需與整體一致
  3. 無圖——Instagram 略過，僅發 Facebook
- **可選**：純文字仍可發布（Facebook），IG 會自動跳過

## 6. YAML Front Matter 欄位說明

```yaml
---
title: ""           # 標題，建議 15 字以內，口語化
slug: ""            # URL 用，英文小寫加連字符，例：bernoulli-principle
category: physics   # math / physics / chemistry / biology / geography / earth-science
tags: []            # 2–4 個標籤，對應課綱單元，例：[力學, 流體]
status: draft       # draft（草稿）→ ready（可發）→ scheduled（已排程）→ published（已發）
publish_date: ""    # ISO 8601，例：2025-07-01T10:00:00+08:00
platforms:
  facebook: true
  instagram: true
summary: ""         # 一句話摘要，50 字以內，用於社群預覽與網站 meta description
cover_chart: ""     # 可選，圖表路徑，例：science/physics/assets/bernoulli-01.svg
curriculum: ""      # 可選，對應課綱，例：國中理化（摩擦力）、國中地球科學（月相）
source_long: ""     # 可選，若本篇是從長文切出的短篇，填入長文 slug
series: ""          # 可選，系列名稱，例：雪地駕駛物理學
series_part: 1      # 可選，本篇在系列中的順序（1 開始）
series_total: 2     # 可選，系列總篇數
---
```

## 7. 發文流程

```
寫稿（status: draft）
  ↓ 自我檢查：字數、結構、語氣
修稿（status: draft）
  ↓ 確認可以發出
改為 ready + 填入 publish_date
  ↓ push 到 main
GitHub Actions 自動偵測 → 發 FB / IG → 改為 published
```

### 長文與短篇的關係

**推薦流程：先寫長文，再切短篇。**

1. 寫完整版長文（500–800 字），確認知識點正確、邏輯清楚，存為 `[slug]-full.md`
2. 從長文萃取 2–3 個核心點，各自寫成短篇（150–250 字）
3. 短篇的 `source_long` 欄位填入長文 slug
4. 短篇社群貼文末尾加一行：「完整說明 → [連結]」
5. 長文可同步發到 GitHub Pages，作為深度內容讓有興趣的讀者自行閱讀

長文本身不一定要發社群，短篇才是社群主力。長文也可以作為寫短篇時的草稿和查核依據。

---

### 系列文章順序規則

- 系列文章的 `publish_date` 必須嚴格按照 `series_part` 排序
- 建議間距：同系列相鄰兩篇至少間隔 **1 天**（讓讀者有吸收時間）
- 發文腳本會檢查：若某篇的 `series_part > 1`，前一篇必須已是 `published` 才會發出
- 排程時建議先把整個系列的 `publish_date` 一起填好，避免漏排

## 8. 分類對應課綱

| 分類 | 目錄 | 對應課綱 |
|------|------|----------|
| 數學 | `math` | 數與量、幾何、代數、統計與機率 |
| 物理 | `physics` | 力學、熱學、光學、電磁、波動 |
| 化學 | `chemistry` | 物質結構、化學反應、酸鹼、有機化學 |
| 生物 | `biology` | 細胞、遺傳、演化、生態、生理 |
| 地理 | `geography` | 地形、氣候、人文地理、環境議題 |
| 地球科學 | `earth-science` | 地殼變動、大氣、海洋、天文 |
